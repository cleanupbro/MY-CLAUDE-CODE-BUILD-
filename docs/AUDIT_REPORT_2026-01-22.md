## Detailed Issue Analysis

### Issue 1: Direct URL Routing Broken

**Problem**: The app uses React state-based routing instead of URL-based routing. When a user directly navigates to `/services` or `/residential`, the app ignores the URL and shows the Landing page.

**Root Cause**: In `src/App.tsx:97`:
```javascript
window.history.replaceState({ view: 'Landing' }, '', '/');
```
This line always sets the initial state to 'Landing', ignoring the actual URL.

**Fix Required**: Parse `window.location.pathname` on mount and set the appropriate initial view.

**Files to Modify**: `src/App.tsx`

---

### Issue 2: manifest.json Syntax Error

**Problem**: Console shows `Manifest: Line: 1, column: 1, Syntax error`

**Impact**: PWA features won't work, console error on every page load.

**Fix Required**: Check and fix `/public/manifest.json` syntax.

---

### Issue 3: tawk.to Widget 404

**Problem**: `https://embed.tawk.to/675e3b97af5bfec1dbe1ad5f/1ietm9umd` returns 404

**Cause**: Either the tawk.to widget ID is invalid or the account was deleted.

**Fix**: Update the tawk.to widget ID or remove the integration if not needed.

---

### Issue 4: Google Fonts 404

**Problem**: Product Sans font files returning 404:
- `https://fonts.gstatic.com/s/productsans/v5/N0c8y_dasvG2CzM7DYSfWDP3iUc.woff2`
- `https://fonts.gstatic.com/s/productsans/v5/N0c8y_dasvG2CzM7DYSf1DP3iUc.woff2`

**Cause**: Product Sans is not a publicly available Google Font (it's proprietary to Google).

**Fix**: Replace with Inter, SF Pro, or another similar sans-serif font.

---

### Issue 5: CDN Tailwind Warning

**Problem**: `cdn.tailwindcss.com should not be used in production`

**Impact**: Slower page loads, potential styling inconsistencies.

**Fix**: The build process should already compile Tailwind. Check if CDN link is in `index.html`.
